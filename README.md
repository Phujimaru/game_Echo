# 🃏 ECHO — Blackjack Skill Battle

เกมการ์ดต่อสู้เล่นหลายคนแบบเรียลไทม์ (ไม่มีเจ้ามือ) — จั่วการ์ดเลข **สุ่ม 1–10** ให้แต้มใกล้ 21 สุดโดยไม่เกิน ใครใกล้สุดชนะรอบแล้วได้โจมตี ใครเลือดจริงหมดก่อนตกรอบ เหลือคนสุดท้ายคือแชมป์

## วงจร 1 รอบ = 3 เฟส
1. **เฟสไพ่** (60 วิ): จั่วการ์ดจนพอใจ + ใช้สกิลได้ แล้วกด **เปิดไพ่** (= พร้อม; ไพ่/แต้ม/สกิลของคนอื่นถูกซ่อนจนทุกคนพร้อม)
2. **เฟสสรุปผล**: เปิดแต้มทุกคน + ประกาศผู้ชนะ (แต้มสูงสุดเสมอกัน → สุ่ม 1 คน)
3. **เฟสโจมตี**: ผู้ชนะเลือกเป้าโจมตี (คำนวณจากตัวละคร+สกิล)
> จบ 3 เฟสแล้วขึ้นแบนเนอร์ **"รอบที่ N"** ไปต่อเอง — ไม่ต้องกดยืนยัน · จบเกมเมื่อเหลือผู้รอดคนเดียว

## กติกาต่อสู้
- **พลังชีวิต**: เลือดจริง 3 + เกราะ 2 — เลือดจริง 0 = ตกรอบ / เกราะฟื้น +1 ต้นรอบ
- **ความเสียหาย**: แต้มน้อยสุด −1 (โดนเกราะก่อน) · ผู้ชนะโจมตี −1 เข้าเลือดจริง (แรงขึ้นตามสกิล)
- **หลอดสกิล** (เต็ม 6): จบเทิร์น +1 · ชนะ +2 · โดนเพราะแต้มน้อยสุด +2 · โดนโจมตี +2
- **สกิล**: พื้นฐาน(2)/รอง(4)/ท่าไม้ตาย(6) + สกิลติดตัว(ฟรี) — data-driven ใน [characters.js](characters.js)

## วิธีเล่น
1. เข้าหน้าเว็บ → กด **เริ่ม**
2. ตั้งชื่อ + เลือกตัวละคร → **เข้าห้องรอ**
3. รอเพื่อน (สูงสุด 6 คน) แล้วกด **เริ่มเกม** หรือกด **เล่นคนเดียว**
4. **จั่วไพ่** เพิ่มได้เรื่อยๆ พอใจแล้วกด **เปิดไพ่**
5. รอทุกคนเปิด หรือหมดเวลา (30 วิ) → ประกาศผู้ชนะ

> เป็นระบบปิด: ไม่มีห้อง/ลิงก์เชิญ ทุกคนเข้าเว็บเดียวกันแล้วเล่นด้วยกันเลย

## สแตก
- **Frontend**: React (Vite) + Tailwind CSS — โฟลเดอร์ `client/` (ธีม ECHO)
- **Backend/Realtime**: Node.js + Express + Socket.IO — `server.js` (เกมเดียวทั้งเว็บ, ยังไม่มีระบบห้อง)

## รันในเครื่อง

### โหมดพัฒนา (dev — แก้แล้วเห็นผลทันที) ใช้ 2 เทอร์มินัล
```bash
# เทอร์มินัล 1: เซิร์ฟเวอร์เกม
npm install
npm start            # ที่ http://localhost:3000

# เทอร์มินัล 2: React dev server (proxy socket ไป :3000 ให้เอง)
npm run client       # เปิด http://localhost:5173
```
เปิดหลายแท็บที่ :5173 เพื่อจำลองหลายคน

### โหมดจริง (prod — build React แล้วให้เซิร์ฟเวอร์เสิร์ฟเอง)
```bash
npm run build        # ติดตั้ง+build client -> client/dist
npm start            # เปิด http://localhost:3000 (server เสิร์ฟ React build ให้อัตโนมัติ)
```
> ถ้ายังไม่ได้ `npm run build` เซิร์ฟเวอร์จะ fallback ไปหน้าเวอร์ชัน vanilla เดิมใน `public/`

## ใส่รูปตัวละคร
วางไฟล์ `1.png` ถึง `10.png` ในโฟลเดอร์ `public/avatars/`
(ถ้ายังไม่ใส่ เกมจะใช้อีโมจิสัตว์แทนให้อัตโนมัติ)

## ขึ้น Render (PaaS)
1. อัปโหลดโค้ดขึ้น GitHub
2. ไปที่ Render → **New +** → **Web Service** → เชื่อม repo
3. ตั้งค่า:
   - **Environment**: `Node`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`
4. กด **Create Web Service** → รอ deploy เสร็จ
5. เอา URL ที่ได้ (เช่น `https://xxx.onrender.com`) ส่งให้เพื่อนเข้าเล่นพร้อมกัน

> Render จะกำหนดพอร์ตให้เองผ่าน `process.env.PORT` (เซิร์ฟเวอร์รองรับแล้ว)

## โครงสร้างไฟล์
```
server.js              เซิร์ฟเวอร์ + เอนจินเกม (Socket.io) — HP/เกราะ/สกิล/ต่อสู้/ตำแหน่ง
characters.js          config ตัวละคร+สกิล (data-driven) + สีตำแหน่ง P1-P6  ← แก้ตัวละครที่นี่
package.json           สคริปต์ start / build / client
client/                React (Vite) + Tailwind — ธีม ECHO
  src/
    App.jsx            คุมลำดับหน้าจอ + รับ roster/positions จาก server
    socket.js          เชื่อม Socket.IO
    index.css          Tailwind + ธีมสี ECHO + glitch/halftone/อนิเมชัน
    data/              avatars.js (อีโมจิสำรอง), positions.js (สี P1-P6)
    screens/           Splash / Setup(ชื่อ+ตำแหน่ง) / CharacterSelect / Lobby / Game
    components/        Avatar / Card / StatBar / PlayerPanel / Button
  dist/                ผลลัพธ์ build (server เสิร์ฟอันนี้)
public/                หน้าเวอร์ชัน vanilla เดิม (fallback ถ้ายังไม่ build)
  avatars/             รูปตัวละคร 1.png - 10.png (วางที่ client/public/avatars ก็ได้)
```

## เพิ่ม/แก้ตัวละคร
แก้ที่ [characters.js](characters.js) ไฟล์เดียว — แต่ละตัวมี `passive / basic(2) / secondary(4) / ultimate(6)`
เอฟเฟกต์รองรับ: `heal, armor, points, shield, draw, redraw` (ใส่เป็น array เพื่อรวมหลายอย่าง)
passive.trigger: `roundStart | win | lose | attacked` — engine ยิงให้เองตามจังหวะ

## สถานะการพัฒนา (ต่อจาก ECHO spec)
- ✅ เอนจินกติกาต่อสู้ (HP/เกราะ/หลอดสกิล/ดาเมจ/ตกรอบ/วนเทิร์นเอง)
- ✅ ธีม ECHO ตามสเปค (พาเลตต์ ม่วง/น้ำเงิน/เทา, HP จุดแดง/เกราะฟ้า/สกิลทอง, glitch, watermark, ฟอนต์ Kanit)
- ✅ ระบบตำแหน่ง P1–P6 + สีประจำตัว, หน้าตั้งชื่อ+เลือกตำแหน่ง, หน้าเลือกตัวละคร+รายละเอียด
- ✅ สกิล 4 แบบ data-driven (basic/secondary/ultimate + passive)
- ⏳ **ถัดไป**: สกิลที่ยิงใส่คู่ต่อสู้ (ตอนนี้เอฟเฟกต์เข้าตัวเอง), รูปตัวละครจริง, ระบบห้อง roomId
